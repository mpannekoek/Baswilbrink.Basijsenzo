# Personas

## Persona 1: Authenticated Admin Content Editor
- **Role**: Trusted shop administrator with existing access to the control portal
- **Primary Goal**: Keep the landing-page text current without editing source code
- **Key Needs**:
  - A simple place to update grouped landing-page text
  - Dedicated pages for opening hours and the taste of the week
  - Confidence that saved changes appear on the public site
  - Protection against accidental overwrites during first-time setup or later edits
- **Constraints**:
  - Uses only the existing control portal access model
  - Does not want a complex CMS or page builder
  - Needs clear save behavior and predictable page structure
- **Success Signal**: Can sign in, edit the relevant text, save safely, and see the updated content reflected publicly

## Persona 2: Public Visitor
- **Role**: Visitor browsing the public landing page for information about the shop
- **Primary Goal**: Quickly read accurate, current text about the shop, opening hours, and featured taste
- **Key Needs**:
  - Up-to-date landing-page copy
  - Clear opening hours and featured weekly flavor information
  - A stable, fast, readable public page
- **Constraints**:
  - Has no admin access
  - Must not see broken content, internal errors, or editing interfaces
- **Success Signal**: Can trust the public page to show current information without noticing the internal management system behind it
